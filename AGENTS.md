# 开发协作说明

本文件记录面向自动化编码助手和维护者的项目内约定。

## 工作原则

- 先确认目标和现状，再修改代码。
- 改动保持小而明确，每一行变更都应服务于当前任务。
- 优先使用现有 MATLAB 脚本入口，不为单次需求新增框架。
- 文档、路径和命令必须与实际仓库保持一致。

## 验证要求

- 修改文档或仓库元信息后，至少检查 Markdown 文件可读、链接路径存在。
- 修改算法、训练或测试流程后，运行 `startup_check.m`，必要时运行 `train.m` 和 `test.m`。

## 仓库边界

- 不删除作业验收需要的 `gesture_model.mat` 和 `data/`。
- 不提交本地 `test_images/`、临时输出、日志或编辑器配置。
