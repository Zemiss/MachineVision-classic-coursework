# 机器视觉技术实验：传统手势识别工作要求

## 一、实验名称

课程设计 Part 1：传统手势识别

## 二、实验目标

使用传统的视觉检测、特征提取与分类算法，在简单背景图像中识别四类手势：

- A
- C
- Five
- V

实验要求不使用深度学习方法，而是基于传统机器视觉方法完成图像预处理、特征提取、分类模型训练与测试。

## 三、实验数据集说明

数据集包含四种手势类别，每个类别包含 50 张背景较为单一的图像。

数据类别如下：

| 类别编号 | 类别名称 |
|---|---|
| 1 | A |
| 2 | C |
| 3 | Five |
| 4 | V |

建议数据集目录结构如下：

```text
dataset/
├── A/
│   ├── xxx.png
│   └── ...
├── C/
│   ├── xxx.png
│   └── ...
├── Five/
│   ├── xxx.png
│   └── ...
└── V/
    ├── xxx.png
    └── ...
```

## 四、编程语言要求

可选编程语言：

- MATLAB


## 五、算法实现要求

应使用传统机器视觉方法完成手势识别流程，整体流程建议如下：

```text
图像读取
→ 图像预处理
→ 手势区域提取
→ 特征提取
→ 分类模型训练
→ 模型保存
→ 测试代码加载模型并识别图像
```

### 1. 图像预处理

可根据数据特点选择以下方法中的一种或多种：

- 图像灰度化
- 图像尺寸归一化
- 图像平滑或滤波
- 图像二值化
- 背景分割
- 形态学操作，如腐蚀、膨胀、开闭运算
- 手势轮廓提取
- 手势区域裁剪与归一化

### 2. 特征提取

应至少实现一种传统特征提取方法，可选方法包括但不限于：

- HOG 特征
- Hu 不变矩
- 轮廓特征
- 形状特征
- 边缘方向直方图
- 面积、周长、长宽比、圆形度等几何特征
- 多种特征组合

### 3. 分类模型

应使用传统分类模型进行训练，可选模型包括但不限于：

- SVM
- KNN
- 决策树
- 随机森林
- 朴素贝叶斯
- AdaBoost

训练完成后，需要将分类模型保存为文件，供测试代码直接加载使用。

## 六、代码提交要求

提交的程序代码中必须包含：

1. 训练代码
2. 单独的测试代码
3. 已训练好的分类模型文件
4. 必要的说明文档

其中，测试代码必须能够独立运行，并满足以下要求：

- 能够加载训练好的分类模型文件；
- 能够读取指定文件夹中的所有 `.png` 图像；
- 能够对每张图像进行手势识别；
- 能够依次打印每张图像的识别结果；
- 输出类别必须为 `A`、`C`、`Five` 或 `V`。

## 七、测试代码功能要求

测试代码运行后，应能对一个文件夹中的所有 PNG 图像进行识别。

示例输入：

```text
test_images/
├── test_001.png
├── test_002.png
├── test_003.png
└── ...
```

示例输出：

```text
test_001.png: A
test_002.png: Five
test_003.png: V
...
```

测试代码不得依赖重新训练模型。运行测试代码时，应直接加载已经训练好的模型文件。

| 是否提交说明文档 | 必须提交 |

## 十二、运行方式示例

### MATLAB 示例

训练模型：

```matlab
run('train.m')
```

测试图像：

```matlab
run('test.m')
```

或在 `test.m` 中指定测试文件夹路径：

```matlab
testFolder = 'test_images';
```

### C++ 示例

编译：

```bash
mkdir build
cd build
cmake ..
make
```

训练模型：

```bash
./train ../dataset
```

测试图像：

```bash
./test ../test_images ../model/gesture_model.xml
```

## 十三、注意事项

1. 测试代码必须能够独立运行，不能要求重新训练模型。
2. 测试代码中应明确指定或允许用户输入测试图像文件夹路径。
3. 图像读取时只处理 `.png` 格式文件。
4. 程序输出应清晰显示每张图像对应的识别类别。
5. 代码应有必要注释，方便检查和复现实验。
6. 若使用特征归一化，测试阶段必须使用训练阶段保存的归一化参数。
7. 若使用随机划分训练集和测试集，应固定随机种子，保证结果可复现。
