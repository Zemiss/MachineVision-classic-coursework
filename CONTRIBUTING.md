# 贡献指南

感谢你考虑改进本项目。项目规模较小，贡献时请优先保持结构清晰、改动可验证。

## 开发原则

- 保持 MATLAB 脚本可直接在项目根目录运行。
- 不引入深度学习框架，除非项目目标发生明确变化。
- 不为单次使用场景增加复杂抽象。
- 不提交本地临时文件、日志、编辑器配置或测试图片目录。
- 修改算法时同步更新 `README.md` 中的流程、特征说明和当前结果。

## 提交流程

1. 确认环境可用：

   ```matlab
   run('startup_check.m')
   ```

2. 如果修改训练、预处理、特征或分类逻辑，重新训练并记录结果：

   ```matlab
   run('train.m')
   ```

3. 使用独立测试脚本验证模型加载和预测流程：

   ```matlab
   testFolder = 'test_images';
   run('test.m')
   ```

4. 提交前检查文档是否仍然准确。

## Pull Request 要求

PR 描述应包含：

- 改动目的；
- 影响到的脚本或文档；
- 已执行的验证命令；
- 如修改算法，提供新的准确率或混淆矩阵。
